# Localization

Rodinka supports Czech (`cs`) and English (`en`). `i18next` and
`react-i18next` own the active language and notify React when it changes.

## Where translations live

The typed application catalogs are in `src/strings.ts`, grouped by feature
(`calendar`, `chores`, `shopping`, `family`, `reminders`, and so on). Both
languages must keep the same structure; `src/i18n/i18n.test.ts` checks this.
Language detection, persistence and locale helpers live in `src/i18n/`.

## Adding UI copy

1. Add a semantic key to both the Czech and English feature sections in
   `src/strings.ts`. Do not use a Czech sentence as the key.
2. Read it through `t.feature.key`. Use a function for interpolation or plural
   forms, for example `t.shopping.activeCount(count)`.
3. Do not create translated module-level constants. Read `t` while rendering
   or while executing the helper so a live language change is reflected.
4. Keep database enums and user-created content unchanged; translate only
   their labels in the UI.

In development, i18next logs missing native i18next resource keys to the
console. The catalog parity test catches missing keys in the typed catalog.

## Detection and persistence

`rodinka.language` in `localStorage` wins. Without it, Czech browser locales
(`cs`, `cs-CZ`, and other `cs-*` variants) select Czech; all other browser
locales select English. Changing the selector in **More / Settings** applies
immediately, persists locally and updates `<html lang>`.

Notification preferences already contain a `locale` field. Reminder copy is
generated from that locale on the server and from the active app language in
the foreground; no database migration is required.

## Adding another language

1. Add its code to `SUPPORTED_LANGUAGES` in `src/i18n/language.ts` and to the
   i18next configuration in `src/i18n/index.ts`.
2. Add a complete catalog in `src/strings.ts` with the same structure.
3. Add the language's native name to the selector in `MoreScreen`.
4. Extend locale detection and `localeFor`, then add tests for detection,
   persistence, pluralization and date formatting.
